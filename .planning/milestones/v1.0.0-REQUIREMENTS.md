# Requirements — v1 Launch-Ready Portfolio
# Archived: SHIPPED 2026-03-05

## Scope

This document covers all requirements for Milestone 1: a publicly accessible, recruiter-ready single-page portfolio.

---

## Functional Requirements

### Setup & Infrastructure

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| REQ-001 | Install and configure Tailwind CSS v3 with PostCSS | Must | Done (01) |
| REQ-002 | Configure Tailwind with a custom design token set (black/white palette, editorial font sizes) | Must | Done (01) |
| REQ-003 | Replace default Vite scaffold (App.jsx, App.css, index.css) with portfolio shell | Must | Done (01) |
| REQ-004 | Set up React Router or anchor-based smooth scroll navigation | Must | Done (01) |

### Navigation

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| REQ-010 | Fixed or sticky top navigation bar with section links (About, Projects, Resume, Contact) | Must | Done (02) |
| REQ-011 | Navigation collapses to a hamburger menu on mobile | Must | Done (02) |
| REQ-012 | Active section highlighted in nav on scroll | Should | Done (02) |

### Hero / About Section

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| REQ-020 | Display developer name in large editorial headline typography | Must | Done (03) |
| REQ-021 | Display one-line role/tagline beneath name | Must | Done (03) |
| REQ-022 | Display short bio paragraph (2–4 sentences) | Must | Done (03) |
| REQ-023 | Section fills or near-fills the viewport on load | Must | Done (03) |

### Projects Section

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| REQ-030 | Display 1–3 project cards in a grid or column layout | Must | Done (04) |
| REQ-031 | Each card shows: project title, short description | Must | Done (04) |
| REQ-032 | Each card shows: tech stack tags (e.g., React, Python, etc.) | Must | Done (04) |
| REQ-033 | Each card shows: link(s) — GitHub repo and/or live demo | Must | Done (04) |
| REQ-034 | Cards are readable on mobile (stack to single column) | Must | Done (04) |

### Resume Section

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| REQ-040 | Embed PDF resume using `<iframe>` or PDF.js viewer | Must | Done (05) |
| REQ-041 | Provide a visible "Download Resume" button linking to the PDF asset | Must | Done (05) |
| REQ-042 | PDF asset stored in `/public/resume.pdf` | Must | Done (05) |

### Contact Section

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| REQ-050 | Display GitHub profile link with icon | Must | Done (06) |
| REQ-051 | Display LinkedIn profile link with icon | Must | Done (06) |
| REQ-052 | Display email address as `mailto:` link | Must | Done (06) |
| REQ-053 | All external links open in a new tab with `rel="noopener noreferrer"` | Must | Done (06) |

---

## Non-Functional Requirements

| ID | Requirement | Priority | Status |
|----|-------------|----------|--------|
| REQ-060 | Mobile-first responsive design — tested at 375px, 768px, 1280px breakpoints | Must | Done (07/08) |
| REQ-061 | Semantic HTML5 elements (`<header>`, `<main>`, `<section>`, `<nav>`, `<footer>`) | Must | Done (07/08) |
| REQ-062 | Keyboard navigable — all interactive elements reachable via Tab | Must | Done (07/08) |
| REQ-063 | No color other than black, white, and shades of gray | Must | Done (07/08) — superseded: indigo accent added in Phase 7 |
| REQ-064 | No gradients, glassmorphism, neon accents, or particle effects | Must | Done (07/08) |
| REQ-065 | Fast initial load — no unnecessary JavaScript bundles | Should | Done (07/08) |
| REQ-066 | Page title and meta description set in `index.html` | Should | Done (07/08) |

---

## Out of Scope (v1)

- Contact form with backend / email sending
- Blog or writing section
- Dark/light mode toggle
- CMS integration
- Analytics
- Animations beyond CSS transitions

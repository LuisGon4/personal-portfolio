<task type="auto">
  <name>Task 1: Install Tailwind CSS v3 and generate config files</name>
  <files>tailwind.config.js, postcss.config.js</files>
  <action>
Run the following two commands in sequence:

```bash
npm install -D tailwindcss@3 postcss autoprefixer
npx tailwindcss init -p
```

The `init -p` command generates both tailwind.config.js and postcss.config.js. Because package.json has `"type": "module"`, the CLI should generate ESM syntax (`export default`). After the command completes, read both generated files and verify they use `export default`, not `module.exports`. If either file uses `module.exports`, convert it to `export default { ... }` syntax (do NOT rename to .cjs — keep the .js extension with ESM syntax).

After verifying ESM syntax, overwrite tailwind.config.js with the final design token configuration:

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,jsx}",
  ],
  theme: {
    extend: {
      fontFamily: {
        sans: ['Inter', 'system-ui', 'ui-sans-serif', 'sans-serif'],
        mono: ['ui-monospace', 'SFMono-Regular', 'Menlo', 'monospace'],
      },
      colors: {
        'off-white': '#f8f8f8',
        'near-black': '#0a0a0a',
      },
      fontSize: {
        '9xl': ['8rem', { lineHeight: '1' }],
        '10xl': ['10rem', { lineHeight: '1' }],
      },
    },
  },
  plugins: [],
}
```

Critical: Use `theme.extend` (not `theme`) to preserve all of Tailwind's default color scale, including `transparent` and `currentColor`. Using `theme.colors` instead of `theme.extend.colors` would wipe the entire default palette.

Leave postcss.config.js as generated (it should already have tailwindcss and autoprefixer registered). Do not touch vite.config.js — Vite picks up postcss.config.js automatically.
  </action>
  <verify>
    <automated>node -e "import('./tailwind.config.js').then(m => { const c = m.default; const ok = c.content.includes('./src/**/*.{js,jsx}') && c.theme.extend.colors['off-white'] === '#f8f8f8' && c.theme.extend.fontFamily.sans[0] === 'Inter'; console.log(ok ? 'PASS' : 'FAIL: config missing expected tokens'); process.exit(ok ? 0 : 1); })"</automated>
  </verify>
  <done>tailwind.config.js exports a config with content paths covering ./index.html and ./src/**/*.{js,jsx}, theme.extend containing Inter fontFamily, off-white and near-black color tokens, and 9xl/10xl font sizes. postcss.config.js exists and registers tailwindcss and autoprefixer.</done>
</task>

<task type="auto">
  <name>Task 2: Replace src/index.css with Tailwind directives</name>
  <files>src/index.css</files>
  <action>
The current src/index.css contains Vite scaffold CSS that conflicts with Tailwind Preflight. It sets `color-scheme: light dark`, `color: rgba(255,255,255,0.87)`, dark background, and blue link colors — all of which violate the black/white design constraint.

Replace the entire contents of src/index.css with only these three lines:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Do NOT keep any existing Vite scaffold rules. The file should contain nothing else — no comments, no :root declarations, no media queries. Tailwind Preflight (included in @tailwind base) handles browser normalization.

Note: main.jsx already imports './index.css' — no change needed there.
  </action>
  <verify>
    <automated>node -e "import('fs').then(fs => { const content = fs.readFileSync('src/index.css', 'utf8').trim(); const lines = content.split('\n').filter(l => l.trim()); const expected = ['@tailwind base;', '@tailwind components;', '@tailwind utilities;']; const ok = lines.length === 3 && lines.every((l, i) => l.trim() === expected[i]); console.log(ok ? 'PASS' : 'FAIL: index.css has unexpected content: ' + content); process.exit(ok ? 0 : 1); })"</automated>
  </verify>
  <done>src/index.css contains exactly three lines: @tailwind base, @tailwind components, @tailwind utilities. No Vite scaffold CSS remains.</done>
</task>
